🏗️ Architecture Overview

Input → Embedding → Positional Encoding → Encoder/Decoder Layers → Output
Key Innovation: Self-attention mechanism allows parallel processing and captures long-range dependencies better than RNNs.

🔍 Self-Attention Mechanism

$$\text{Attention}(Q, K, V) = \mathrm{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$
  • Q (Query): "What am I looking for?" - represents the current position/token
  • K (Key): "What information is available?" - all positions in the sequence
  • V (Value): "What is the actual information?" - the content to be retrieved
  • √dk scaling: Prevents softmax saturation in high dimensions
Intuitive Understanding: Think of attention as a soft database lookup:
  1. Query: Your search term
  2. Keys: Database indices
  3. Values: Actual data stored
  4. Attention weights: How relevant each database entry is to your query
  1. Linear Transformations: Input embeddings → Q, K, V matrices
  2. Attention Scores: Compute QKT (how much each position attends to others)
  3. Scaling: Divide by √dk to prevent gradient vanishing
  4. Softmax: Convert scores to probabilities (attention weights)
  5. Weighted Sum: Multiply attention weights by values

🎯 Multi-Head Attention

Why Multiple Heads? Single attention head might focus on one type of relationship. Multiple heads can capture:
  • Syntactic relationships (subject-verb agreement)
  • Semantic relationships (word meanings)
  • Positional relationships (nearby vs distant words)
  • Different abstraction levels
$$\text{MultiHead}(Q,K,V) = \text{Concat}(\text{head}_1, ..., \text{head}_h)W^O$$
where headi = Attention(QWiQ, KWiK, VWiV)
  • Typical setup: 8-16 heads for base models, 32+ for large models
  • Dimension per head: dmodel / h (e.g., 768/12 = 64 for BERT-base)
  • Parameter sharing: Each head has its own WQ, WK, WV matrices

📍 Positional Encoding

The Problem: Transformers process all positions simultaneously → no inherent sequence understanding
$$PE(pos, 2i) = \sin\left(\frac{pos}{10000^{2i/d_{model}}}\right)$$
$$PE(pos, 2i+1) = \cos\left(\frac{pos}{10000^{2i/d_{model}}}\right)$$
  • Sinusoidal Encoding: Deterministic, relative distances, extrapolation
  • Learned Positional Embeddings: Trainable, better for fixed-length, no extrapolation
  • Rotary Position Embedding (RoPE): Encodes positions as rotations in complex space, better extrapolation (LLaMA, GPT-NeoX)
  • Alibi: Adds position-dependent bias, very good extrapolation

🏗️ Transformer Variants

Encoder-Only (BERT-style): Bidirectional context, masked language modeling, classification/NER/QA
Decoder-Only (GPT-style): Causal masking, autoregressive generation, text generation/completion/dialogue
Encoder-Decoder (T5-style): Separate encoding/decoding, cross-attention, translation/summarization/seq2seq
  • Choose encoder-only for classification and full-context tasks
  • Choose decoder-only for text generation and streaming
  • Choose encoder-decoder for translation, summarization, and complex outputs

🔧 Key Architectural Components

  • Layer Normalization: $$\text{LayerNorm}(x) = \gamma \frac{x - \mu}{\sigma} + \beta$$ (Pre-norm: more stable, Post-norm: original design, RMSNorm: used in LLaMA)
  • Feed-Forward Networks: $$\text{FFN}(x) = \max(0, xW_1 + b_1)W_2 + b_2$$ (SwiGLU, GeGLU, typically 4x wider)
  • Residual Connections: $$\text{output} = \text{LayerNorm}(x + \text{SubLayer}(x))$$ (enables deep networks, stability)

🎯 Interview Q&A

Q1: Why did transformers replace RNNs for many NLP tasks?
A: Parallelization, long-range dependencies, training efficiency, scalability, transfer learning.
Q2: Explain the intuition behind attention weights.
A: Relevance scoring, soft selection, context aggregation, learned associations.
Q3: What are the computational bottlenecks in transformers?
A: O(n²) attention, memory bandwidth, sequence length scaling, solutions: sparse/linear/flash attention.
Q4: How would you modify a transformer for very long sequences?
A: Sparse/sliding/ hierarchical attention, memory mechanisms, retrieval augmentation.

    position = torch.arange(seq_len).unsqueeze(1)

                        -(math.log(10000.0) / d_model))
    pe = torch.zeros(seq_len, d_model)
    pe[:, 0::2] = torch.sin(position * div_term)  # even indices
    pe[:, 1::2] = torch.cos(position * div_term)  # odd indices
    return pe
